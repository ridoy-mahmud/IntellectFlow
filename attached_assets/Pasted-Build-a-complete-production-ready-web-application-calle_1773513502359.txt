Build a complete, production-ready web application called "NEXUS AI" — an Autonomous Multi-Agent Orchestration Platform where multiple specialized AI agents (Researcher, Writer, Coder, Analyst, Designer, QA Tester) collaborate autonomously to complete complex projects. The app must be visually stunning, fully functional, and feel like a futuristic mission control center for AI agents.

=== TECH STACK ===
- Frontend: Next.js 14+ with App Router, TypeScript, Tailwind CSS, Framer Motion for animations
- UI Components: shadcn/ui component library
- State Management: Zustand for global state
- Real-time: WebSocket connections for live agent activity streaming
- Backend API Routes: Next.js API routes that connect to a Python/FastAPI backend
- Database: Supabase (PostgreSQL) for persistence
- Authentication: Supabase Auth (email/password + Google OAuth)
- AI Integration: OpenAI API (GPT-4) for agent intelligence, with support for multiple LLM providers
- Charts/Visualization: Recharts for analytics, ReactFlow for workflow graph visualization

- Primary background: #0A0A0F (deep space black)

- Use gradient text for headings: linear-gradient(135deg, #6366F1, #06B6D4)

=== REAL-TIME SIMULATION ENGINE ===
Since we need the app to feel alive and dynamic even without a real Python backend connected, implement a sophisticated simulation system:

Create a file: lib/simulation-engine.ts that:
1. When a mission is launched, it starts a simulation loop
2. Every 2-5 seconds (randomized), it generates a realistic agent activity message
3. Messages follow logical project progression:
   - Researcher starts first: searches, finds sources, compiles brief
   - Writer receives brief, creates outline, writes sections
   - Coder analyzes requirements, writes code, runs tests
   - Analyst processes data, creates charts description
   - QA reviews all outputs, provides feedback
   - Agents can "reply" to each other naturally
4. Progress bar increments realistically
5. Tasks move through kanban board automatically
6. Token counters increment
7. Occasionally (10% chance), an agent "requests human review" triggering the intervention UI
8. Mission completes after simulating all workflow stages (takes about 3-5 minutes of simulated time)
9. Use realistic, domain-specific messages. For example:
   - Researcher: "I found 23 relevant academic papers on transformer architectures. The most cited work is 'Attention Is All You Need' (Vaswani et al., 2017) with 89,000 citations. I'm now cross-referencing with recent 2025 publications..."
   - Writer: "Based on the research brief, I'm structuring the article into 5 main sections: 1) Introduction to Multi-Agent Systems, 2) Architecture Patterns, 3) Real-World Applications, 4) Implementation Guide, 5) Future Outlook. Beginning with the introduction now..."
   - Coder: "Initializing Python project structure. Creating virtual environment. Installing dependencies: crewai==0.28.0, langchain==0.1.9, fastapi==0.109.0. Setting up project scaffolding..."

Store simulation state in Zustand so it persists across page navigations.
